这类问题通常对行、稀疏 和 稠密的情况。。。各存在一个暴力算法。。然后分块讨论选择哪个算法就行了。。。

const int N = int(1e5) + 9;
VI P[N]; int n, Q; LL ans;

bool Find(int x, int y){
return (*lower_bound(ALL(P[x]), y)) == y;
}

int main(){

#ifndef ONLINE_JUDGE

Q = sqrt(RD(n))+1; REP(i, n) P[RD()].PB(RD());
#define Pi P[i]
REP(i, N){
SRT(Pi); Pi.PB(INF); if (SZ(Pi)>Q){
REP(ii, i) ECH(it, P[ii])
if (Find(i, *it) && Find(i, *it-i+ii) && Find(ii, *it-i+ii)) ++ans;
}
else{
REP_2(r, l, SZ(Pi)-1, r)
if (i-Pi[r]+Pi[l] >= 0 && Find(i-Pi[r]+Pi[l], Pi[l]) && Find(i-Pi[r]+Pi[l], Pi[r])) ++ans;
}
}

OT(ans);
}

Problem E.
虽然一眼看出是 2D/1D 的 DP。。。
状态时 dp[位置][区间] 。。。但是转移并不是很显然。。。。

主要原因还是状态设计的并不清晰 。。。。


dp[ i ][ j ] ：
表示所有区间右端点不大于 i 的都考虑过，
且最长链为 j 且结尾恰为 i 的方案数。。

那么我们枚举 k 转移到 dp[k][j+1]。。。

..j.....k...

新加入的区间分是否跨越 j 讨论。。
。不跨越 j ：那么所有左端点在 (i, k] 右端点在 k 的区间都是可以加入的。。（又因为至少需要加入一个。。所以是 2^(k-i) - 1）。
。跨越 j：左端点在 [1..i]，右端点 (i, k] 。。。（共2^[i*(k-i)] 种 ）。。。

最后统计的时候。。枚举 i。。。再考虑跨越 i 的情况就可以了。。。
。。。。这样做事没有重复的？。。

[4,  6]
[3,  5]
[2,   4]
[1,  3]

上面这种情况会不会被计数两次？。。。。
不会。。。。只在 dp[5][2] 计数一次。。。因为这里状态里的长链是最小表示。。（右端点尽可能小）


注意预处理 pow2 否则会 TLE。。
