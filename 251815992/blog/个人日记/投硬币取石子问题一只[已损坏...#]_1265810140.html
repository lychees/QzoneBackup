[题目简述]

这是一个博弈问题...题目是这样子...

有n个石子...  爱丽丝和鲍勃两个人轮流投硬币...爱丽丝先. 并且如果投到正面则要取出一个石

子......取出最后一块石子的一方获胜...

[进一步的条件]

爱丽丝和鲍勃两个人有一个"投币神技" (Skill of throwing a coin)...

当他们心里投念着硬币的一个面时...意念的力量会使得成功的概率提高 p%。

[简化模型的分析]

解决这题还是从简单的情况开始...我们先忽略第二组条件...只考虑彼此投币的概率都是50%的情况... (外面打雷了...+. +) 并且在简单一点..(简单到最简单的..不能再简单的情形...)考虑只有一块石子的情况...

50%      25%        12.5%   ..->>越来越小..

Alice        Bob         Alice..  Bob..Alice.

考察玩家Alice的胜率...

50%+ 50%×50%×50%+50%×50%^4...

这个是一个无穷几何级数...a0=50%..公比等于 25%...普通的几何级数公式是这样..

[排公式中...]...

（上面好像有点小问题...我下标标的是a0。。但是又是从1开始的...算了反正不影响阅读...）

考虑到公比小于1所以数列会越来越小...所有可以忽略掉最后 q^n...于是就是。。。


代入 a0 和q 可以计算出结果...

为了求 Bob 同学也可以用同样的方法...但是也可以利用前面计算的结果直接用一减去即可...

[不断的增加参数...]

接下来考虑n=2...如果还是按照前面的方法的话...则要枚举出所有的结果...但是这一次就不像前面那样简单了因为是无限的...而n=2的时候又不是一条直线便于前面我们那样分析....我们只是简单的列出了一些结果...可以看到大概是这样一颗统计树...左路是投到正面...

之所以画得往右下角倾斜是因为这颗树的“左高度”不可以超过2..所有树叶都是在左高度为2地方...但是因为可以一直这样往右边延伸延伸延伸...所以和前面那个数列一样也是个无穷的物体...

接下来稍微研究一下这颗树...黑色的层代表 Player II Bob.行动的回合...白色层代表Player I 爱丽丝行动的回合...而且我们发现每一层每一个圆圈所代表的“事件”发生的概率...是一样的..但是具体计算起来还是比较讨厌..因为每一层的“圆圈”数目比较难以统计..唔..或者说..具体左高度的“圆圈”数目比较难以统计 ..[1]

正确的思路应该是看看能不能利用一下前面的结果找到递推公式...我们设

F[n]表示取n个石子时爱丽丝获胜的概率...

G[n]是取n个石子时鲍勃同学获胜的概率...

很自然的可以知道F[n] + G[n] = 1...所以 G 数列可以不要...

接下来我们需要寻找可以通过F[1]计算出F[2]的方法（或者...我们应该说通过F[n-1]计算F[n]的方法...不过这里我们将更多的注意力放在1推2..）

。。。

。。。

但是我发现不可以....即便知道了F[1]的值...还是无法计算出F[2].。。原因是我们对F[1]获胜时刻的状态所收集的信息不够用...起码得知道...F[1]结尾的时候是停在Alice处..还是Bob处呀...

额..等等...这个我们好像知道...F[1]最后可定是Alice结尾....

等一下...让我要把思路理一理...F[1]最后一定是在Alice处结尾...那么我们不知道的是什么...?

我们不知道的是下一次以Bob为先手的一个 F`[1] ...

...嗯...但是F`[1] = F[1] ...所以...

F[2]=(1 - F[1])  × F[1] ...

然后略做处理得到...

F[n]=(1-F[n-1]) × F[1]  ...

不过这样还少一种...额...就是开始输掉的那种...也就是分F[1]和(1 - F[1])讨论...应该是...

F[2] =  (1 - F[1]) ×F[1]  +  F[1] ×(1 - F[1])

F[n] =  (1 - F[n-1]) ×F[1]  + F[n-1]×(1-F[1])

接下来加入"投币神技"   ... 按照原题的叙述设置为 p 和 q ..

继续考虑...这一次同理可得..

F[1] = p / (1-(1-p)*(1-q));

F[2] =  (1 - F[1]) ×F[1]  +  F[1] ×(1 - F[1])  ??

这一次F[2]的计算公式就有问题了...让我们回忆一下先前的定义..

F[n]表示取n个石子时爱丽丝获胜的概率... 它是由.两部分所加得..分别对应..

F[1]是爱丽丝获胜和F[1]的时候是鲍勃获胜的情况...额...这样我们的定义就出现问题了...

我们已经默认F只能是爱丽丝获胜的情况...但是在这里为什么可以把它作为另一种鲍勃获胜的情况来对待？而且在前面我们就是这么做的...原因是在于...这一次两个人需要区别对待了...因为掷硬币的概率是相同的....我们最好在这里再停一下...思路比较乱..。

其实是一共有四种情况...

A : 爱丽丝先手爱丽丝取到一枚硬币

B:  爱丽丝先手却让鲍勃取道下一枚硬币

C: 鲍勃先手鲍勃取道一枚硬币...

..D也是对称的...4种情况是因为考虑了两种不同的因素..分别是

是谁先取子...和..谁取到了下一枚硬币...

有了A,B,C,D我们就可以很轻松的写出递推公式来了...因为这一次的思考顺序也已经很明朗了..

F[1] = A;

F[2] = F[1] × D     +        ( 1 - F[1]  )  ×  A

再记下这几个两之间的关系...这里面有几个是互斥关系...写一下...

A + B = 1 ;  C + D = 1;

A和D是什么关系？

（这个问题可以加分...A和D在前面一问中是互斥关系...因为那时

A=C,B=D 但仅仅是数值上相等...逻辑上并不一致...而在此之前我出现的思维混乱是因为他们数量相等把我麻痹了...）

F[n]  =  F[n-1] ×D  + （1 - F[n-1]）× A

...参见程序[1] ...

但是这个是错的...

在后来的交流中我发现我忘记了一个重要的问题...就是每个人不一定要投"正面"...也就是说..每个人都有了一定的改变命运的能力...它们会根据场上的情况选择接下来的决策....

那么什么时候我们会主动去投“反面”呢？！

答案是在情况不对的时候...

我们去投反面相当于主动把接下来的选择权交给对方而不去让当前的 n 减小一...

FootNote:

写这篇的时候发现这个也不是很难统计...具体是要看你怎么想...我开始的时候想错了...

在统计时..应该暂时把其它左高度的节点补齐..这样子总数是2 的 n 次方...而按照左高度（或者右边也可以）顺序排下来的话刚好是一行杨辉三角（Pascal Triangle）的数字...也许这是解这题的另一种思路...但仍然是受限于无限...？！@
