采访问题：

> 1、我们了解到您热衷于参与开源项目，您的 GitHub 也有近两百个 Repositories，一直致力于支持开源。您有没有统计过自己大概参与了多少开源项目？第一个参与的开源项目是什么，是怎样的契机让您参与到其中？另外，最近参与和研究的一个开源项目或者方向又是什么？

A：说来惭愧，其实那些 Repositories 大多是 star 别人的项目，我自己应该没有什么在开源界有影响力的项目。第一个参与的开源项目是我打 ACM 期间的 LaTeX 模板和代码库，说起来大概那可能也是影响力最大的项目，经常我还能看到有后辈私信告诉我说当初是看我的代码学的算法。另外几个引用比较高的是几个 Dapp 的范例程序，很多人是参考我们提供的教程，开始编写和设计以太坊和 EOS 的 Dapp。最近有参与的开源项目可能是 [Defi.WTF]( https://github.com/carboclan/Defi.WTF-Official-Website )，我在活动的网站里植入了一块基于以太坊的哈伯格税广告牌。

> 2、2018 年，您在离开谷歌之后，创办了自己的公司仙女座科技 Andoromeda，开始推动区块链的发展。想问一下，在您的创业过程中，与原先作为工程师相比，有哪些心态上的变化？在自己开公司以后，对开源项目的看法或者态度，有没有发生过一些转变？
>

A：心态上的变化到没有什么特别的，主要是看待事物的角度更加全面了。区块链行业是一个非常混乱和复杂的领域，每一天都会有新的东西冒出来，混乱和复杂，每一天都会有新的东西冒出来，币圈一日，人间一年并不是一句 meme，而是这个行业的真实写照。而从之中过滤掉噪音，遴选出真正有价值的想法和技术就会是非常重要的工作。得益于身边有一群最优秀的小伙伴，我感到过去一年里我的知识和眼界都前所未有的拓宽了。当然相应的代价就是实际写代码的时间变得更少了，因为总是能找到 ROI 更高的事情要去做。对于此，之前 Gitcoin 的 CEO @Owocki 在推上有过一段辩论，认为并不是创业公司的 CEO 到了 A 轮之后如果还在写代码，那么这个公司就一定不行，虽然我也想认同他的观点，可我知道就现阶段看来那只是一个美好的梦。

https://twitter.com/owocki/status/1186304886817710080

> 3、本次的 2019 开源年会上，您将带来一场演讲报告，期待您的分享！对于开源的发展，您觉得区块链和开源技术，它们之间有着怎样的联系？比如有说法称，「区块链是开源的接班人」，或者「开源是区块链发展的核心」等，您对这些观点又是怎么看待的？

这是一个很有意思的话题，我这周刚刚 clockworkpi 的创始人 Hal 有过长达几个小时的讨论。Hal 认为目前两大群体之间的交集还比较小，对此我并不认同。

我的观点是，我们正处在某个变革的零界点。Linux 中国上个月推送过来自 Red Hat 的 Bilgin Ibryam 的一篇名为 [区块链能如何补充开源]( https://mp.weixin.qq.com/s?__biz=MjM5NjQ4MjYwMQ==&mid=2664615147&idx=2&sn=c5d4e4ad4c0e7f004e224773a0a2ef0f&chksm=bdcef7ad8ab97ebbe897db28e5a6346574f6e67a86b24a5549ddf0e1d15a5f2761288e324eeb&mpshare=1&scene=22&srcid=&sharer_sharetime=1572094923439&sharer_shareid=2529a9a000929104b90737a98bce8930#rd ) 的文章，实际上搜索过去可以看到原作者是写了两篇对偶的文章，时间大概是去年：
- [What blockchain can learn from open source]( https://opensource.com/article/18/11/what-blockchain-can-learn-open-source )
- [How blockchain can complement open source]( https://opensource.com/article/18/9/barter-currency-system )

上面提到的 @Owocki 今年在 Devcon5 的演讲主题叫做 The Future History of the Open Internet

- [📹Bilibili, Devcon5, The Future History of the Open Internet, Kevin Owocki]( https://www.bilibili.com/video/av70818578/ )

如果你仔细对比 Amir Taaki 2017 年在 HCPP 上的那个演讲，你会发现基本上大家都是 「Connecting the Rabbit Dots」（Owocki 在 slides 里用了一个更形象的说法，叫做「盲人摸象」）。

而上面提到的这些论述的共同点是，都会连接 Richard Stallman 的 Free Software Movement，到 Linux，再到大教堂与市集，再到比特币。

Bitcoin is the first time humans have organized at scale without hierarchy.
https://twitter.com/ericsavics1/status/1186823577435164673


对此我认为，问题的关键是，我们能否从一个更广阔的视角审视比特币这个项目本身，能否发现并推广比特币的众筹模式。我们上周在台北 Crosslink 对开源和区块链的关系此也稍微有一段讨论。
https://medium.com/@feihutang/crosslink-recap-design-pattern-build-your-first-profitable-dapp-and-smart-contract-66f767ee3a39


我的基本观点是，未来我们会 One Repo One Token，甚至 One Person One Token，真正做到 Tokenlize Everything 让 Token 贯穿软件从孕育，到立项，到众筹，到开发迭代，到最终付梓上架，到最后分润结算（不仅仅结算法币，还包括 Credits 和 Reputation）的生命周期的每一个环节，公司的规模和周期都会变得更小更简单。而这就是未来我所寓言的 —— 协作的扩展。

另外下面有一些关于您个人的问题，如果不方便的话，您可以选择不回答。我们在出稿前会和您进行确认。

4、您的升学经历和后来在圈子里知名，都和竞赛有关。对于国内的这类竞赛，您认为有哪些让人惊喜的地方，还有哪些可以改进的地方？对那些注重实践的同学，您有哪些建议？

中学时期我是有一点多动症的，参加过的竞赛很多，学校里日常的文化课程其实很难吸引我，有挑战的事情才能让我全情投入。如果没有参加这些竞赛的话，可能很多时间就浪费掉了。我参与最多的是 OI/ACM 竞赛，他们于我最大的魅力，一个是基本没有考纲，日常的训练有一个 Build 自己的过程，可以看到自己不断解决自己不会的问题，另一个就是比赛的公平性和竞技性。遗憾的是，这两个赛事，最近都遇到了一些问题。前者遭遇了 [改名风波]( https://zhuanlan.zhihu.com/p/82653178 )，不过我国庆节的时候跑去少儿培训机构客串了一节讲师，看起来现在环境已经控制住了。后者 [更惨]( https://www.zhihu.com/question/351414766/answer/865336559 )，而且如果你是这个圈子里的人根本不知道我都经历了什么，关于此，其实我也是最近才想通。

> 劣币问题在所有使用铸币的人类文明中都会出现，只要由官方强行规定法定货币的流通权，就不可避免会出现劣币驱逐良币的现象。那么，如何解决劣币驱逐良币的问题呢？解决的办法只有一个。只是这个办法是任何政府都不想采纳的。
> 这个办法是：开放铸币权，允许所有的人铸造货币；但在同时，也放开民间的选择权。
> —— 《中央帝国的财政密码》
>

和奥派的观点一样，如果一个货币有问题，我们就分叉她，事实上分叉的自由或许是整个区块链的生态中最高的自由。比赛也是如此，得道者多助，资源会自然的像更好的地方倾斜。而现实中我们也是这么做的。


> 5、您个人很喜欢 Cosplay 和 Lolita 装扮，也经常在活动中以女装的形象出席，这破除了很多人对程序员的固有印象。这种鲜明的个人风格，对您的个人工作有没有意料之外的影响? 另外，对于大众对工程师的固定化印象（比如格子衫），在您看来有哪些好的/不好的地方吗？

作为国内出道最早的女装程序员之一，其实我女装形象示人的时候是非常少的。。。
而且因为日常熬夜，缺少必要之保养，现在已经成为时代的眼泪了。
不过我还是坚持工程师还是要靠自己的代码作为自己的名片。
不过我还是崇尚开放、多元与自我表达，这些也和 Open Source 的价值观一致。
参见 Github 上影响力很大的那个 CrossDress 的 Repo 吧。
https://github.com/komeiji-satori/Dress

我觉得我们应该更多的聚焦再一些 [晚辈]( https://www.zhihu.com/people/vxst/activities ) 身上，她们各方面做的也都比我要更好。


> 祝您一切安好！
