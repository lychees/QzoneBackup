十分有趣。。。

设状态 (i, x, y) 表示物品 i 到物品 n，先手 x 钱，后手 y 钱时是否必胜。。



Critical Point：
1. x + y + s_{i-1} = a + b。。 说明 y 时多余的。。
2. (i, x) -> (i, x+1)。。单调性。。


结合以上两点。。说明我们需要注意的是。。。
f[i]：先手至少需要多少金币可以必胜。。。


于是不难得到 O(n2) DP 。。。

DWN_1(i, n, 1){
f[i] = s[n]-s[i-1];
FOR_1(j, i+1, n){
checkMin(f[i], max(s[j-1]-s[i-1], (A+B-(f[j]-1)-s[i-1])));
}
}

。。。。。不难优化到 O(n)。。

int ff = s[n]; DWN_1(i, n, 1){
f[i] = ff - s[i-1];
checkMin(ff, max(s[i-1], A+B-(f[i]-1)));
}
