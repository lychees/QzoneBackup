1. DP 很简单。。关键求两个点之间能否看见。。。
2. O(n3) 暴力也可以过。。。（而且跑的飞快。。。）
。。枚举视点，砍掉相邻的两条边形成的区间（这里同多边形的方向有关。。 数据的多边形都是逆时针给出。 。 ）
之后直接线段相交即可。.（不需要区分规范相交）

3. O(n2logn) 做法。。
。枚举视点。。

3.1  方法一：
。。对每个线段按照到视点的距离排序。
set 维护当前可见的角度区间。。
支持插入一个区间。。（插入区间可能会引起区间合并。。）

3.2 方法二：
对所有点按照极角排序。。之后用 set 去维护所有线段。。。
。对于每组线段拆分成插入事件和删除事件。。用所有线段在当前方向的距离作为关键字。。
。。注意这类问题的关键字大小是会随着扫描线的推进而改变的。。。（nlogn 线段相交）。。
。。但是相对关系仅在关键点（插入和删除的时刻）发生变化。。。
