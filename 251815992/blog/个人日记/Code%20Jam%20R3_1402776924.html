Tower 的攻击是按照顺序的，对于每个 Monster，我们有两个决策，杀/不杀。。。
。。。如果不杀的话。。我们并不是去枚举去攻击的对象，而是把攻击次数攒起来。。。
这样就是 0/1 背包。。背包容量 = 攻击次数。。。因为有负容量的物品，因此必须使用滚动数组。。。

a[i] = ceil(hp, Q) ： 选择不杀时产生的容量
b[i] = ceil(hp%Q?hp%Q:Q, P)-(a[i]-1); 杀时花费的容量。

RD(P, Q, n); REP(i, n){
int hp; RD(hp, v[i]);
a[i] = ceil(hp, Q),
b[i] = ceil(hp%Q?hp%Q:Q, P)-(a[i]-1);
}

p = 0, q = 1; FLC(dp[p], -1); dp[p][1] = 0;

REP(i, n){

swap(p, q); FLC(dp[p], -1);

#define u dp[q][j]

REP(j, M) if (~u){
checkMax(dp[p][j+a[i]], u);
if (j-b[i] >= 0) checkMax(dp[p][j-b[i]], u+v[i]);
}
}

int res = 0; swap(p, q); REP(j, M) checkMax(res, u);
OT(res);
}
}
