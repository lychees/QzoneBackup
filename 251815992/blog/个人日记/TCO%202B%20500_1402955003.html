随机两个数 x，y。。Mrs. S 知道两个数的和 S，Mrs. P 方知道两个数的积 P。。。
。。
Mrs S：“我确定你不知道 S 是多少。。 ” （不作死就不会死？）
Mrs P：“现在我知道 S 是多少了。”

问 [l, r] 范围内符合上述条件的   S 的和。

0.
。。P 何时不知道 S 是多少？ 。。
。。 P 仅在 P 是质数的情况下。。能确定。S 的值。。（1+P)。。。



-1.
S 如何知道 P 的值一定是合数？。。。
S = x + y。。。 不失一般性 设 x <= y。。。
。。如果 x > 1。。那么 P 一定是合数了。。
。。只要保证当 x = 1 时，P 也是合数即可。。
而此时 P = S-1。。。
也就是 S-1 是合数。

。。。P 如何能在这种情况下确定 S 的值？
。。现在 P 的手中又多了一个条件。。（S-1 是合数。。。
。。。枚举所有可能的 x，y。。判断是否有唯一解。。就把单点询问的问题解决了。

const int PMAX = int(5e6) + 9;
const int N = int(5e6) + 9;
VI P; bitset<PMAX> isC; LL S[N];

void sieve(){

FOR(i, 2, PMAX){
if (!isC[i]) P.PB(i);
for (int j=0;j<SZ(P)&&i*P[j]<PMAX;++j){
isC[i*P[j]]=1, S[i*P[j]+1]=i*P[j]+1; if (!(i%P[j])) break;
}
}

FOR(x, 2, N){
for (int y=x;(LL)x*y-1<N;++y) if (isC[x+y-1])
S[x*y+1] = 0;
S[x] = S[x] + S[x-1];
}
}

class SumAndProductPuzzle {
public:
long long getSum(int A, int B) {
if (!SZ(P)) sieve();
return S[B]-S[A-1];
}
};
