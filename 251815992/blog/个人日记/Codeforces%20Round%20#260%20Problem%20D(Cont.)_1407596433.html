这个题的 splay 做法还是很有趣的。。。。


。。。首先注意到我们直接 splay 维护区间的话。。无法处理询问。。


解决方法是。。。 对于每个值再建一个 splay 。。。

我们用 rt[k] 表示关键字为 k 的 splay 的根。。（0 表示整个序列的）

————————
于是询问只要对只要在 rt[k] 中，找到对应的区间求 sz 就可以了。。。
。。这一步相当于 select 的一个 variation 。。。
。。。我们需要每次用 rk[0] 中 rank 来定位。。而不是本身的 sz。。。
。。。因此询问的复杂度是 log^2n





——————————
考虑如何应付修改。。。

struct node{
static node *NIL; node *c[2],*p,*q;
int sz, ky;
}

。。
对每个结点再加一个 *q 指针。。。用来定位它在 rk[ky] 中的位置（在 rk[ky] 中。。就定位其在 rk[0] 中的位置。。）
于是修改操作就可以写成。。

y = select(b-1, z = select(b+1)), x = ry, ry = NIL, y->upd(), z->upd(); node *xx = x->q;
y = select(a-1, z = select(a)), y->setr(x), y->upd(), z->upd();
z = xx->succ(), y = xx->pred(); y->splay(rt[xx->ky], z->splay(rt[xx->ky]));
ry = NIL, y->upd(), z->upd(); ins2(xx);


也就是先处理比较容易的 rk[0] 。。。然后用 xx 记录被移动的结点在 rk[k] 中的位置。。。
。。再用前驱和后继把这个结点找到删去。。。最后再把它重新插入到新的位置。


http://codeforces.com/contest/455/submission/7406849



————————

这个方法似乎可以用 “动态标号” 优化成 O(nlogn) 的。。。
