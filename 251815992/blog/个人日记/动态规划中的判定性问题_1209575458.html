砝码称重
设有1g，2g，3g，5g，10g，20g的砝码各若干枚（其总重 ≤ 1000g），要求：
输入：
a1   a2   a3   a4   a5   a6(表示1g砝码有a1个，2g砝码有a2个，......20g砝码有a6个)
输出：
Total=N (N表示用这些砝码能称出的不同重量的个数，但不包括一个砝码也不用的情况)
输入 样例 ：1  1  0   0   0   0
输出 样例 ：Total=3，表示可以称出1g，2g，3g三种不同的重量
首先我们将注意力转移到这样的一道题目上面来...读一下题目看看我们都知道些什么...
若干砝码的质量～要求出用它们所能称出的不同质量的个数～
一个可见的思路是...咳咳～～当然是我们亲切的穷举方法～～
枚举每一颗砝码的状态（加或不加）...
但是...本题（其总重 ≤ 1000g），考虑到极限状态上可能有2的1000次方种组合方案
（= -） 穷举法显然会 超时~~~
不得不寻找另外的思路...
动态规划
我们考虑放入第i个砝码时，只需要依次考虑增加1到a 块砝码i到i-1次时砝码的质量产生情况...
也就是新的状态总是有前一个状态所决定...这满足了我们动态规划表格法的性质....那么开始...设计我们的数据结构...
const num:array[1..6] of integer=(1,2,3,5,10,20); { 砝码 的重量序列 }
type  int=integer  {lychees的习惯= -，冒似每次都可以少打不少字，而且也很容易的切换成longint} 嘻嘻
var     a:array[1..6] of int; {这个用来记录每颗砝码的数量}
dp:array[1..1000] of boolean;{**这里不同于我们以往的动规最优化问题,这里是布尔型变量**}
ans:int;tRNA:int{记录状态转移} {另外开一个记录一下最后的答案好了}

阶段i:分析第i块砝码。
状态j:枚举第j种质量。
决策k:for k:=1 to a do 增加k*num 块砝码～

我们发现状态j需要枚举当前已经存在的质量...而我们需要通过从1～ans枚举dp中为真值得情况...
一个小的优化是再开一个数字 weight:array[1..1000] of int;记录不同的质量情况；
我们将ans=weight[0] 于是可以合并这两个数据结构。

至此算法的核心部分以及数据结构的设计已经完成...~很容易写出代码～
{初始化}..weight[0]:=1;weight[1]:=0;
for i:=1 to n do
for j:=1 to weight[0] do
for k:=1 to a do
tRNA:=k*num +weight[j];
if not dp[tRNA] then
begin
dp[tRNA]:=true;
inc(weight[0]);
weight[weight[0]]:=tRNA;
end;

输出：
Total=N (N表示用这些砝码能称出的不同重量的个数，但不包括一个砝码也不用的情况)
writeln(weight[0]-1){减去质量为0的情况就好了}
