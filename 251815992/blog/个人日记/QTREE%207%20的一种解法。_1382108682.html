You are given a tree (an acyclic undirected connected graph) with n nodes. The tree nodes are numbered from 1 to n.
Each node has a color, white or black, and a weight.
We will ask you to perfrom some instructions of the following form:
0 u : ask for the maximum weight among the nodes which are connected to u, two nodes are connected if all the node on the path from u to v (inclusive u and v) have a same color.
1 u : toggle the color of u(that is, from black to white, or from white to black).
2 u w : change the weight of u to w.

http://acm.hdu.edu.cn/showproblem.php?pid=4645


这个题涉及动态树的子树求 min 操作。。不妨先考虑子树求和。。。QTREE 6.。。
。。。
。。给定 n 个结点的黑白树。。初始颜色给定。。支持以下操作
0. 询问与某个结点颜色相同的连通块的大小。。（就是 dfs() 进去染色。。。看和他颜色相同的有多少个点。。。
1. 单点颜色 toggle


我们对每个节点建立两个辅助节点。。 分别连接它的所有黑孩子和所有白孩子。。。
每个节点的父亲指针指向它的父亲对应颜色的辅助节点。。 这样 Toggle 的复杂度就和结点的度数 无关了。。
。 。 于是每次修改 Cut、Link 两次即可。。。

这样就是每个结点要分出三个结点（其本身（实）。以及黑 / 白。(虚）。虚结点不存结点本身的信息。。。
。。进一步我们每个结点建两 黑/白 两个实结点就行了。。
这样每次修改。只需要 Cut、Link 各一次。。。（但修改的时候要改两个了。。。


对于 QTREE 6 我们每个结点维护其子树的和。。。每次 Cut / Link 都是只对这个结点到根的路径产生影响。。
所以用标准的 LCT 就可以解决。。

——————

再回到 QTREE 7 。。。存在一种非常好写的处理子树的方法。。。每个结点我们直接存一个 multiset<int> s; 。。
保存其子树中（所有虚孩子）的信息。。。


那么 upd 操作现在写作。。

inline void upd(){
w1 = max(l->w1, w0, r->w1, *s.rbegin());
}

。。。注意 acs() 的时候维护 s 即可。。

inline node* acs(){
node *x = this, *y = NIL; do{
x->splay();
if (y != NIL) x->s.erase(y->w1);
if (rx != NIL) x->s.insert(rx->w1);
rx = y, x->upd();
y = x, x = px;
} while (x != NIL);
return y;
}


然后 cut / link 的时候也需要维护。。

void cut(node *x){
x->acss(); splay();
x->s.erase(w1); p = NIL;
}

void link(node *x){
x->acss(); splay(); //assert(p == NIL);
p = x; p->s.insert(w1); p->upd();
}

————————————————

几乎就是在 LCT 的基础上做很少的改动。。。。复杂度大概是 O(nlog^2n)。。。
但是实践中确是最好写。速度也最快的方法。。。 。。。（四叉动态树居然就被这样虐爆了？。）
