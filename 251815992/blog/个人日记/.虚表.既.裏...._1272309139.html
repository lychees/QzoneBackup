花了一晚上时间修复了网站一个Bug..

[修好了] 网站坏掉了….。。啊啊啊

April 26th, 2010 · `Alice Scarlett., · Diarist No comments

… 晚上吃过晚饭以后开始修 Google Analyse （结果截至 GMT+8 2010年4月26日 20：31为止我所有的数据还是显示 0 ..）… 虽然我估计这个站的流量只有个位数，但是除了我生拉硬拽来的几个同学，应该还有什么别的人起码路过一下这里吧，我是很好奇他们是从哪儿来的的…

结果，哎，好倒霉啊啊… 我的博客就成了现在这个样子了囧…

… 稍微理一下思绪…主要的情况是因为我拖了一个 Google Analyse Dashboard 的 Widgets 到侧边栏…结果侧边栏和页脚还有标题栏的位置都乱掉了…囧… 然后更糟糕的是居然我把它拖回去以后居然还不行，我试图依靠我仅有的那些可怜的 HTML 和 CSS 知识去修复它，可是立刻陷入了巨大的窘境… 我感到很茫然很无助… 为了不让大家看到令我伤心的结局，我把主题暂时换成了一个适合用手机访问的屏幕…
…
…

GMT +8 Apr 26th 22:23

修好了… 是我自己的错… 正在想为什么会出现这样的问题…

- -01:54

…

- -01:57

我最后却是修好了额Google Analyse...额，发现访问量不是我之前所想的个位数。。而是十位数 。。。

嗯，我简单写一下，我开始的想法全都是错的..我的网站分页出现了混乱，并且其中的 一篇 甚至套住了所有的文章，页脚的部分也跑到了岁上面，为了解决这个问题，我花了很多时间结果学习了很多WordPress和PHP的知识...真晕....虽然说最后的原因很简单，我在写某一篇的时候不小心在最后的地方多加了一个半开的<div>标签，结果这个标签居然之前下去以至于影响到整个博客的布局....

....

本来上面的这一篇是放在网站里的...

但是...

我后来想了一想，觉得这一篇除了对我自己以外...对别人一点营养也不会有，（而我又舍不得删）所以就把它放QQ空间里来了..

....

..

等一下！这个好像有点不太吧，放在你自己的网站里嫌不好，就把它放在QQ空间里来给我们阅读？。。。你。。你你。。也太看轻这里了吧？。。。

。。。

嗯，听我说完。。。

半个月前。（嗯，刚好15天。） 我打完仙四在写感想的时候呢 以后开始建 这个网站 ，并且最终整个网站仍然很不明朗。但是这个博客居然奇迹般的每天都有更新，并且在短短两个星期之内居然有如此的肉实。。实在令其他的某人和某人都该党不可思议。。。#额，，不过这个也很好解释。。毕竟我以前也写QQ空间的习惯。。。额。。。对文字的驾驭能力也有一点心得。（咳咳，是您打字比我们快吧...Orz...）

今晚遇到了一个初中同学..名字叫  圣龙剑客——张晓晨...嗯，自然我会把自己的网站发过去给他看一看额。。结果自然是很失望，他大概没看多久就把网站关了。。。。

然后。。。我强忍住自己无比的不爽情绪（你知道人家为这个站花了多少心血。。##额额）。。然后“虚心”像他请教还有什么要改进的地方。。。

嗯，当然我觉得自己是有收获的...

这些天里呢我一直都在想一个问题，这个博客对我究竟有多大意义？。。。嗯，表面看起来好像是这样，在博客建好的第二天，我开始从 Pascal 转 C++ 了，并且这几天也写了很多好题。于是我们不禁要问，这个博客是为了我写程序，或者说的更进一步一些，是为了我以后的 ACM 所做的么？。。。

结果他在第一次访问这个页面的时候就看到了许多自己不熟悉的语言，以为我的网站是英文写的。。结果就离开了。。。离开了就不知道去哪里了。。（额，我还想以来她多认识一些人呢 > <）...

另外则是您怎么看待这个空间以后的命运？（还有百度这边）在你已经用Word Press搭建了这个博客之后呢？....

嗯， 但是有一点我还是不确定的....

在自己修复网站的那一段空白时间里，我也做了一些其他事情。嗯，比如我重新整理了一下自己的PC机，从桌面到C盘D盘。（整理好看了许多，嘻嘻，有点Mac Book的感觉...）。。。#

然后了，是我放弃了使用 Dec C++ 写程序，而全面换成 Code Blocks （额，一个好处是在每次编写完成须以后不需要加白痴的 system("pauze") 或者 while(true) .. 而且编译速度快了一些。。。。其实当然好处一定不止这些，看不懂括号内容的请无视。）

“.当....作....懂得删掉....自己文章里的文字...时，那么这篇一定...。”

（当然这样的过程对我来说通常很难... 而且糟糕的是我居然忘记了这句话的出处和原句，如果你知道的话，一定告诉我。）

... 于是我把这篇........“删掉”了....

访问自己博客的无非是两类，嗯，当然可以分的更细。。但是我这里只考察两种。。

1. 第一部分会在我的网站里逗留很久，他们会去使用分类和标签云，回去想阅读更多文字。（通常这部分人是我同学。。。囧）

2. 第二部分，每篇文章看完标题发现没有什么有趣的东西。然后关掉。

。。。。（抱歉，我不得不承认这是大部分。。。）

看到 安全卫士360 不断的写什么 云查杀呀 ..杀毒呀什么的。。。然后心里想。。 自己的用户群连IE内核都不知道是什么。。 他们会知道什么是 云计算(Cloud Caculate)？不过仅仅这个名字已经让我感到非常激动想去用这个软件了（实际的情况更糟）。。。

现在，在我的心里，有一个更宏伟的想法存在，但是他的本质和   “云” 是一样的。。。

....

继续。。我必须想办法让我的网页打开的更快，但是毫无疑问自己在以后必然会安装更多的插件甚至用更多的JavaScript和Flash.... 在网站坏掉用了一个手机界面代替的时候，我确实感到了更快的速度是多么重要，  也许简洁和迅速是网上冲浪运动员最喜欢的。

但是在这方面。。。我不会去做丝毫的让步。。。

那怎么办。。。

我的想法是不要让首页呈现那么多文字。。。嗯。。如果每一篇文章只给出标题如何？嗯，但是这样也不太好。。。怎么办？可是现在这样前10篇“全文”展开。。。和“打开”一个LInk进去看又有什么分别？。。。那我侧边的分类和标签云又有什么用？。。。。。

我的想法是。。部分显示。并且按时间。。时间。。。递减。。。。再。。递减。。

（最先写好的第一篇文章全文显示，后面2篇展示一半，后三篇展示三分之一，后5篇只有标题和第一段文字。。。如何？）

。。。。再着就是画风。。。。我对我现在的“水墨画风”感到很惊异。。但是看起来真的是这样。。。。#

我需要两个主题。。不是八个。。。白天，夜晚。。绿色的春天，宁静的夏天，谷香中的秋天和一个雪白的临近极圈的世界。。。（2×4 = 8）

。。。然而，我不能去写 8 份不同的CSS。。。。

。。。。怎么办？

我需要拟合自然的角度。。。。嗯。。是的。。。实际上我只写 6 份。。（并且，这样看只是少些了两份？呵呵呵呵呵？） 白天，夜晚，春天，夏天，秋天和冬天然后通过混入进行CSS叠加。。。然后形成。。。明亮的春天，夜晚的春天，炎阳，夜晚的夏天，两分秋天。。炉火堆与夜晚的冬天。。。！（ 通过组合。。。。我可以即由此轻松得到完全随时间随季节变化我所需要的各种不同主题。。。嗯，那就是我想要的啊！。）

。。。。继续。。。

....在博客建好的制度按时间里，我确实认识到了许多新的东西。。。（当然不完全只有新的网页设计知识呀，画画呀，Gmail 呀什么的）

一、把演算法傳輸到別人腦中的失真率很高。演算法是一個動態的東西，光是以靜態的文字和圖片無法傳達足夠的訊息。我想必須輔以圖片輪播、動畫、互動式Applet，才有辦法提升傳輸效果。可是這些工作，每一項都是大工程，我想我現在的情況是做不來的。如果能組一個團體，專門做這些事情，那不是很好嗎？

二、學習演算法是為了解決問題。本站整理了為數可觀的演算法，可是卻沒有提到演算法能做哪些應用，是美中不足的地方。現在這個世界十分依賴電腦，如果我們學了這些演算法，再發揮一點想像力，想想看日常生活中有那些事情可以用這些演算法來計算，那一定很有趣。

三、這是我人生中一個很重要的問題。想請問哪裡有那種可以不用盯著電腦、不必熬夜和加班，但是又適合程式設計師去做的工作？

。。。这段话引自 这里 。

。。。。例如，我接触并认识了一位台湾同学，（他也是学计算机的。。。）（从我开始惊讶于他的网站的美丽到我了解到他的一些想法.... 虽然我以前也接触过一些台湾人，但是还是完全不同。。。）

。。。

。。

我QQ不一定在线，（特别是写代码的时候我会关掉全部。。）

而在线也不一定不隐身。。。。

。。。。

现在，我可以自己画出我想要的图了。。。因此我不再需要“黄钻”。。。

。。现在，在我拥有了自己的服务器之后。。我可以像这样放各种很难找得到音乐而不再担心空间商或者网络硬盘什么的了。。。因此。。。。

。。。。

我不再需要“绿钻”。。。。。

。。。。（例如现在正在播放的这首 “夜雀之歌”。。。我可以在以后。。借助这个服务器而让这里也变得更有价值和意义！！！！）

最后，欢迎来到 www.税子龙.com 的神秘世界... 你知道我想说什么，就如同我。。？..
