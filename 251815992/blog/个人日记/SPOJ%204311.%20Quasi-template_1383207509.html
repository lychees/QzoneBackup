llh 历史的眼泪。。

后缀数组+平衡树+RMQ+并查集+扩展KMP，O(N(logN)^2)
如果确定类模板长度，那么相同的子串在后缀数组中必是连续一段
易知在不同长度下，这样的连续段（称为平台）最多只有N个，每个对应长度在某区间的一些相同子串
对于每一个平台
把里面的后缀按编号排列
那么子串（类模板）长度至少要是相邻后缀位置差的最大值，这可由子平台平衡树合并实现
对于前面得覆盖的空缺，设第一个后缀为x，则可取的长度集合为union{[i-1,i-x+LCA(1,i)]|i>=x}。离线x从大到小处理，可以通过平衡树（或线段树）实现（每次增加一些区间的长度）
对于后面得覆盖的空缺，可以先二分，再判是否可以覆盖末尾空缺。先求逆串扩展KMP，然后RMQ
写了好久，总是写错。总觉得有更好的方法？
TLE，优化不足
优化1：使用O(N)的后缀数组
优化2：合并平衡树时采用启发式合并与有序表合并中较优的一个，快1倍

Brief description:

给定一个字符串。。。我们定义它的一些子串为 quasi-template。。。如果可以用这个子串复写若干次后。。无空隙的覆盖掉原串。
。。（允许前面、后面多出来一块。）。。
求该字符串的 quasi-template 的数量。。并输出其中最短的 quasi-template。。（如果有多个最短的再输出字典序最小的。。）

。。考虑如何判断一个子串是否是 quasi-template 。。我们可以用 KMP 求出其原串中所有的 occurrence。。。
。。只需判断其中最远的两个 occurrence 距离是否 <= 该子串的长度即可。。

。。然而我们不能枚举所有子串。。。于是我们考虑使用 SAM。。来枚举结点所代表的子串集合。。
。。我们期望从结点中维护出一些信息。。以帮助我们作出某种遴选。。。。。。。

（例如在 hdu 3518 中。。。我们维护出了每个结点最早和最迟的 occurrence 。。。并用这两者的差。。。
。。。来作为该结点。。新的上界。。。）
res += max ( 0 , min ( len [ u ], r [ u ]- l [ u ]) - len [ p ]);

。algorithm on string 中所述的 occurrence 信息。。。（是否 / 总数 / 最早（远） / 所有）
。前面三种只要简单的 SAM-DP 就可以了。。。第四种显然可以　　 平衡树 + 启发式合并。。

void upd(){
dd = 0;
if (l == NIL) ll = ky; else ll = l->ll, checkMax(dd, max(l->dd, ky-l->rr));
