主题: GGJ2019SHJH【流浪地球】

以下内容为临时编写，没有经过深度校对和排版。内容如果有疑问，请微信联系。

这是我参加过的最爽的一次gamejam。
我是从2014年的时候开始参加GGJ，并且在第一次的时候认识了现在我司的CEO小岛美奈子。
这是我第3次参加GGJ，并且也是唯一次提前完成，并且完成了我们对于游戏期望，并且测试没有恶性bug的！我们这组的队员都十分给力！两位程序大佬和另一位策划都是来自国内某知名二次元游戏公司，实力非同小可。这次的主题是What home means to you ?我们最初设想了各种身份各种状态下“家”对于一个人的意义，但是在进一步的脑暴中，我们一致认为在过年之前出这种题目，国内可能很多都会和春节联系在一起，光是描述个人的情感无法体现出全中国乃至全世界的人民们对于家的统一认识。我作为一名科幻迷，最近一直在向周围的人安利即将上映的电影《流浪地球》，我最为一名科幻世界超过10年以上的老读者，对于这部作品我是印象极其深刻的，因为带着地球一起流浪这样的桥段其浪漫程度实在是令我钦佩，我一直都觉得大刘是一个非常浪漫的理工男，因为他还写出过“云天明买下一颗星星”这样的桥段，实在是非常浪漫的事情。
我和团队立马就认为应该从将我们的视角上升到全人类的层面：流浪地球！流浪地球就是我们一直在寻找的主题，一场关于全人类协力保卫家园的故事，一场关于人类生死存亡的浪漫歌剧。
在游戏的过程中有诸多内容并没有符合原著设定（原著没有地球加减速和射击），甚至没有符合物理规律（太阳表面的曲率应该在画面中看起来更像是一个平面而不是一个弧线）
但是这正是GGJ的魅力所在：一切都为了游戏服务。我们没有足够的时间去做各种考证，但是我们能够做出来一个足够有意思且流程完成的有趣的游戏。

我们在制作这款GGJ作品的时候，我们心目中的对象是陈星汉的《风之旅人》。如何在短时间内提供一段心流体验使我们首要考虑的目标，为了达成这个目标，我们在设计的时候有几条隐形原则：
  • 能不用文字就不用文字，能用画面表达就不要现场解说（事实上我们做到了，现场试玩的时候我们没有进行游戏解说。）
  • 玩家同时需要控制的只能够2个或以内的要素（游戏中控制）
  • 核心要素要简单明了，在游戏开始的10秒钟中一定要让玩家可以领会到游戏的目标和规则（比如左边的太阳就是代表了危险，飞来的岩石本能需要躲开等等。）
这些原则是我们作为游戏人对自己的某种不成文的约束，也可以说是一种执念吧。
----------------
游戏中的设定：
地球有一个能量值的设定：
  • 地球加速&射击陨石都会消耗能量，
  • 陨石被击碎后会掉落能量（原作中地球发动机是靠烧石头（重聚变）来驱动），
  • 不加速和接到能量都可以回复能量。
地球被陨石撞击的时候会有被击退的效果，
太阳会间歇性产生日冕，并且会提前给出火苗预警，地球需要及时躲开。
当地碰到屏幕左边的太阳时候，就会gameover。
---------------
感悟
  • GGJ并非是一个比赛，活动的目的是产出有意思的游戏，如果使用公司做项目的那种模式显然是会浪费很多时间。我在现场就观察到了一组公司团，有制作人带队，最开始是想要尽量简化，但是最后居然是产出了一个编辑器，甚至连游戏本体还没有完成，非常可惜，如果有一周的时间进行开发，一定可以做出很好的作品。
  • 游戏要边做边测试，不可以做完再测试。我观察到很多组开发者都是在做完游戏之后没有时间进行测试，演示的时候出现了诸多恶性bug。因此有一个良好的架构意识是作为有项目经验的开发者的优势
  • 简单就是美，策划一定要提炼出来你想要表达的内容中最核心的部分，所有的机制都要为这个核心服务，不要做多余的事情。
  • 一个看起来清晰美观的文档可以省掉很多的沟通成本，同时也可以为您的开发留下文字记录。
  • 在软件的应用方面不要说不可能：我作为一名策划在现场临时掌握了AU，甚至现场缩混了我们的游戏音效和BGM。
  • 游戏中最好可以有彩蛋，这是在路演中为你加分的内容。当然没有彩蛋也不会影响正常游戏。
  • 不要试图让玩家同时处理多个事情
  • 不要为了增加游戏的哲学性意义而故意拔高理解的难度，包括操作层面和表现层面。
  • 最好可以调动玩家的某种原始情绪（比如在流浪地区中调动玩家的“使命感”“紧张情绪”“垂死挣扎”的感觉。）

我一直在找背景音乐，本来想用原野追逐（星际穿越）的原声，但是并下载不到，所以只能使用钢琴音乐，精心挑选了一段出来，关卡的总共时长要控制在 2 分钟左右，所以音乐差不多要配合这样。

我想到了把 Ragerage 这篇诗朗诵给同步播放，本来只是剪辑了一小段出来，但是后面持续的钢琴曲显得非常的单调。我尝试了使用各种方法，最后还是采用了我将所有的英文版本的台词用中文读出来，我自己全程配音，这样用中文应该是更好的，毕竟大家可以听得懂。而且全程配音应该也是一个加分项。

没想到我今天很多时间都是在做音乐上面了。

我发现 AU 我从来没用过，但是到了这个份上了，拿出手就会用了。。。。是我以前太高估难度了。


我今天一直纠结在这个速度的显示条，因为之前想的是飞船要一直加速度，然后达到一定的宇宙速度后可以脱离太阳，但是实际游戏的时候并不能明显感受到速度的变化（需要更多细节的处理）我的设计是背景需要持续不停加速度的，但是实现出来的只是在 boost 的时候会有一个速度线的效果。

并且达到一定的速度可以通关这个目标是难以理解的，首先就是在 UI 上非常不好设计，不知道用什么样的 UI 可以高大上的暗示玩家这个进度条代表的是急速条，并且只要加速到最右侧就可以过关。

最后想了半天，想明白了，这个过关的条件根本就不应该是速度，因为太阳膨胀是有极限的，飞船只需要在被吞噬之前到达终点就可以了。

之前思考加速度是因为被流浪地球的设定给思维困住了，觉得就应该不停吸收石头来加速，但是忘记了同时还有生存的要素，所以达到一个特定的速度这个目标（第三个显得次要的主要目标）会被忽视。所以减少到达一个速度的设定，而是坚持一定的时间（玩家并感觉不到到底需要坚持多少时间）。这样减少了一个需要关心的内容，专注在如何保持自己不会死掉这件事情上就可以了。

UI 的设计迎刃而解，直接做成终点和位置的距离显示就好了

今天下午被这个加速度的 UI 至少耗了我 2 小时的时间，，，，


和这些大佬们组队最大的感受就是想法可以很好的实现了，不会每次都做不出来，而且大家真的都是可以依赖的人。和岛以前组队的时候每次都只能使用各种曲线救国的方式，做的东西也非常不入流，但是快乐还是有的。

和我们一队的美术小弟弟，对于我们策划和程序讨论的时候涉及到的各种和制作相关的内容时候表现出非常惊讶，因为这样的讨论在没有经历过真真实制作环境的人来说肯定是难以理解的，他们不知道移动一个物体是要给物体施加一个力来实现的，不知道做游戏需要考虑到各种物理情况下的合理性，要保证玩家不会出戏。
