正难则反！。。。

我们可以得到 。。 。。当询问的位置集合是 s 是。。
哪些字符串是不可以分辨的。。。

而这个可以通过如下的 DP 实现！！
\ f[ 询问的位置集合 ] = 可以分辨的字符串集合/

REP_2(i, j, n, n) if (i != j){
int s = 0; REP(k, m) if (S[i][k] == S[j][k]) s |= _1(k);
f[s] |= _1(i);
}

DWN(s, _1(m), 0){
REP(i, m) if (_1(s, i)){
f[s^_1(i)] |= f[s];
}
}



先得到极大识别集合。。在传递到子集。。。

剩下的工作就比较 trivial 了。。。

REP(s, _1(m)){
int step = count_bits(s) + 1;
REP(i, m) if (!_1(s, i)){
LL d = f[s|_1(i)]^f[s];
ok[step] += count_bits(d);
}
}

DB ans = 0; REP_1(i, m) if (ok[i]){
DB t = ok[i] * i;
REP_1(j, i-1) t *= j;
FOR_1(j, m-i+1, m) t /= j;
ans += t;
}

ans /= n;
OT(ans);
}

说一下最后统计答案。。。 实际上这里的 ok[i] 是恰好通过一个 i 步的状态可以识别出的字符串的数目。。。
。。。由于要求恰好 i 步。。所以最后一步实际上是固定的。。。

。。。这样总的排列组合数。。就是
\frac{i-1)!}{\Pi_{i=m-i+1}^{m}}
