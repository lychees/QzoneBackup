给定一颗 n 个节点的树，定义 f(a,b) 为树上 a, b 两点通路上的最长边的长度。。
最大化 min(f(i, p[i])) | 1<=i<=n。。要求每个数 i 在 p[] 中不得出现超过 x[i] 次。



【解法 1 贪心 + 并查集】

对边从小到大删除。。。判断删除的边构成的联通块中的点是否可以全部连向联通块外的点。。
。。如果不合法。。。那么当前的边必须出现在路径中。。因此答案就是这条边的权值。

http://codeforces.com/contest/444/submission/7106897


【解法 2 网络流 + 边分治】
——————
。首先二分答案。。然后可以网络流判定。。。
。瓶颈在于。。 边的规模是 O(n2) 的。。

——————
于是考虑用边分治建立辅助结点。

原本是这样。。
s x y t

考虑跨越了边分治中的一条边。。。
e
X  Y

X, Y 是左右两颗子树。。x, y 分别是子树中的某个点。。

我们对每个结点记录 mx[u] 为下降到这个点，所经过的最大权值。
那么 x->y 的条件是。。max(mx[x], mx[y]) >= 当前二分的答案。。。

。。显然我们可以建立两个辅助结点来解决。。

void Link(VI &X, VI &Y){
int e = ++F.n; ECH(x, X) F.add(*x, e, INF);
ECH(y, Y) if (ms[*y] >= L) F.add(e, n+*y, INF);
e = ++F.n; ECH(x, X) if (ms[*x] >= L) F.add(*x, e, INF);
ECH(y, Y) F.add(e, n+*y, INF);
}


。。。然后在对称的处理下 Link(Y, X) 即可。。。。
