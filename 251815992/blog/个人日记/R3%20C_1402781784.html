Problem C. Crime House

...。。你在一个大厅的入口蹲点。。你知道按顺序发生的 n 个事件，每个事件形如
E id：标记为 id 的人进入了大厅。
L id：标记为 id 的人离开了大厅。

。。。id = 0 表示这个人带了一个面具。。你无法确认这个人是谁。
。如果这些情报相互矛盾。。那么说明该大厅一定还有其他出口。。输出 CRIME TIME。。
否则输出结束时大厅最少有多少。。。。你不知道开始时大厅里有多少人。。


————————
。。。这个题出的相当之赞。。。比赛时一直纠结于各种错误贪心。
。。。。反而完全没有考虑到。。更加靠谱的网络流。。。




先不考虑 id = 0 以及大厅出始可能有人的条件。。

我们来看一组简单的数据。。。
E 1
...
E 1
...
E 1

显然。。在这种情况下。。第 1 个 E 1 事件必须在第 1 段 “...” 中被一个 L 1 事件。匹配。
。。第 2 个 E 1 事件必须在第 2 段 "..." 中被一个 L 1 事件匹配。。。
。L 的情况对称。。而 id = 0 的情况。。无非就是多建几条边。。。。。


s -> E -> L -> t

因此主要就是。。带下界网络流、、、但是这个图十分特殊（二分图）。。。根据 7k+ 的说法。。。我们发现可以直接用费用流模拟。。。
。。。方法是对所有有下界的边。。费用设置为 -1 。。。以便于优先把这些边匹配到。。
。。如果一个 E 事件没有被匹配。。那么会导致答案 + 1.。。如果一个 L 事件没有被匹配。。那么我们可以再开始补上。
。。。为了大厅最后剩余的人尽可能少。。显然就是匹配的数目尽可能大。。最后就是求最小费用流。。

。。如果费用不对那么无解。。否则答案是 |E| - max_flow() 。。。

这种做法 Large 需要跑 2.5 mins 。。。
