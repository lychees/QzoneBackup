相对匹配模型：

给定一个文本串，和一个模式串。
。。只要窗口内， 子串之间的相对大小关系 和 模式串一样，就认为匹配成功。（例如，1 5 4 和 2 9 7 ）。。

http://wikioi.com/problem/2529/

CEOI 2011 Day 1 Match：
。。。模式串是一个排列。求所有相对匹配的位移。



namespace KMP{
void get_pi(const char P[], int n, int pi[]){
for (int i = 1, j = pi[1] = 0; i < n; pi[++i] = j){
while (j && P[i] != P[j]) j = pi[j];
if (P[i] == P[j]) ++j;
}
}
} using namespace KMP;





。。。观察 KMP 算法。。。显然我们只需要修改里面的比较函数。。。
。。如何修改？ 。（。一种比较弱的方法是用平衡树维护 窗口。。求 rank()。。）

。继续思考。。。实际上。。我们只需要判断 最小的大于 P[i] 的字符的位置。和 最大的小于 P[i] 的字符的位置。。
。。这两对大小关系就足够了。。。 我们分别记作 a[i] 和 b[i]。。。


namespace KMP{
bool fit(const int P[], int i, int j){
return P[i-a[j]] <= P[i] && P[i-b[j]] >= P[i];
}

void get_pi(const int P[], int m, int pi[]){
for (int i = 1, j = pi[1] = 0; i < m; pi[++i] = j){
while (j && !fit(P,i,j)) j = pi[j];
if (fit(P,i,j)) ++j;
}
}
} using namespace KMP;


由于题目里给出的是排列。。。我们可以逆序用双向链表十分方便的求出 a[i] 和 b[i]。。。
个人认为这个模型相当之 Nice 。。。。
