<h2>Problem J. Tri-war</h2>
<h3>Brief description:</h3>
。。给定 $$n$$ 个结点的一棵树。。每次询问三个点 $$u_0, u_1, u_2$$。。。
。。。点 $$v$$ 被 $$u_x$$ 支配 。。如果 $$\forall y \not = x, d(u_x, x) < d(u_x, y) $$。。
。。。问这三个点所在的支配集的大小。。。

<h3>Analysis:</h3>
<code>倍增祖先</code> <code>LCA</code> <code>分类讨论</code>

<h4>两个点的情况：</h4>
。先把要准备的模板铺好。。显然就是。。。。。（。<a href=" http://www.shuizilong.com/house/archives/codeforces-round-140/ ">Codeforces Round #140</a> 和 <a href=" http://www.shuizilong.com/house/archives/spoj-1486-the-ants-in-a-tree/ ">SPOJ 1486. The Ants in a Tree</a>

。。先考虑 2 个点的。。设函数 <code>up(u, k)</code> 表示 <code>u</code> 结点向上 <code>k</code> 个位置的结点。。。
。。那么通过【倍增祖先】我们就可以将中点找到。。。之后根据路径的奇偶性讨论一下即可。。。

最后要么是。。。
<pre>u1  --- m --- u2 </pre>
（这种情况存在着一些 <code>灰色结点</code>。。（既不属于 <code>u1</code> 也不属于 <code>u2</code>。。
要么是。。。
<pre>u1 --- m1 --- m2 --- u2</pre>

。。。复杂度 $$O(log(n))$$。。。（对于有边权的情况。。复杂度好像仍然是 $$O(log(n))$$ ？。。可以一边二分一边向上跳？）

。。。合理的使用 <code>swap</code> 以提高对讨论的抗性。。、、。。
（本题中 <code>swap</code> 的情况目测会茫茫多。。我们直接使用 <code>ans[]</code> 来记录答案。。）

[cpp]
void gao(int x, int y){
int z = lca(x,y), dx = dep[x]-dep[z], dy = dep[y]-dep[z], dd = dx+dy;
int d = dd/2; if (d>=dx) swap(x, y), swap(dx, dy); int m = up(x,d);
if (dd&1) ans[x] = sz[m], ans[y] = n-ans[x];
else ans[x] = sz[up(x,d-1)], ans[y] = d==dx?sz[up(y,d-1)]:n-sz[m];
}
[/cpp]
