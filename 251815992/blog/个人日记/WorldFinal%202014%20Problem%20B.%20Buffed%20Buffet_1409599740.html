题意：
【凸完全背包】
n 种物品，背包的总容量为 m。。。物品分为两类。离散的 / 连续的。
对于离散的物品，有每份物品的容量，初始价值，和价值的衰减速率。。（/ 份）。。。
对于连续的物品，有每份物品的初始价值，和价值的衰减速率。。（/ 容量）。。。

在背包范围内，最大化所取物品的总价值。

好题。。。我做这种题感到十分吃力 && 极不适应。。。
。。。显然离散和连续是两个相对独立的子问题。。。

先讨论连续的。。（因为比较简单）。。

显然可以贪心每次选取价值最大的。。。。。。然后计算最大的衰退到次大的时间。。。
。然后将最大的物品和次大的物品【合并】成一个新的物品。。（价值的衰减速率会相对变小）。。。。。

具体：

sort(ALL(C), greater<PII>()); C.PB(MP(0, 0)); int ii=0;DB vv=0; REP(i, SZ(C)-1){
if (!C[i].se) vv = OO; else vv += (DB)1./(DB)C[i].se;
DB dd = (C[i].fi-C[i+1].fi)*vv, d = min(m, dd);
z += d*C[i].fi - d*(1./vv)/2; m -= d;
if(!sgn(m,0)) break;
}


这里：
C[i].fi：当前物品的 初始价值 ...（也是每轮迭代，我们 【合并】 后物品的价值。。 ）
C[i].se： 当前物品， 单位容量，价值的衰退速率为
（为了和习惯一致。。这里我们不妨把容量理解为 “时间”。。）
vv：表示下降单位价值，所需要的时间。
(C[i].fi-C[i+1].fi)*vv ：到下个阶段，所需要的时间。。。


首先排序 （O(nlogn) 。。。
。。然后我们不难用上面的过程。。得到每个 [0..m] 的每个容量。。我们用连续的物品所能获得的最大价值。。。
。。。在最后我们枚举分在两个 sub-problem 的容量即可。。
