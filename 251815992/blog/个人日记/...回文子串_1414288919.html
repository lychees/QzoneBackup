对于长度为 n 的字符串 S ，其 不同 的回文子串数量是 O ( n ) 的已经是熟知的事实。

为了找出所有不同的回文子串，一个经典的方法是： 利用 Manacher’s algorithm 求出每个中心的最长回文串长度，按照从长到短的顺序，每次把串的头尾同时去掉一位，利用Hash合并重复的串。因为总数是 O ( n ) 的，所以时间复杂度是 O ( n log n ) 。

因为经典的算法依赖于Hash，感觉不太优美。下面介绍一个确定性的算法，时间复杂度是 O ( n | Σ | ) 。其中 Σ 是字符集。

建立一个自动机 A ，每个状态 u 代表一个回文子串 u （下文将不区分状态 u 和回文子串 u ）。状态 u 往状态 v 有一条标记为 α 的边，当且仅当 v = α u α 。

类似于后缀自动机，对于每个回文子串 u 有对应的出现位置的集合，按照后缀自动机的符号记为 r i g h t ( u ) 。 注意到如果 v 是 u 的后缀（同时也是前缀），则 r i g h t ( u ) 是 r i g h t ( v ) 的真子集。 与后缀自动机的right树一样，我们从 r i g h t ( u ) 连向最小的包含它的 r i g h t ( v ) ，称为后缀链接 s u f 。

如果已经得到长度为 i 的前缀的自动机，现在正在加入在 ( i + 1 ) 的字符 c 。 因为 i , s u f ( i ) , s u f ( s u f ( i ) ) , … 是所有以 i 结束的回文串，只要暴力在其中找到最长的两端都是 c 的即可。计算新的后缀链接可以类似地暴力。
