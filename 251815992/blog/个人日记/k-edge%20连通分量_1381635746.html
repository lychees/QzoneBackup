顺便重新打一下 Stoer_Wagner 的 板子。。。

Stoer_Wagner：每次用 quasi-prim 随机搞出一个 s-t 最小割。。如果它是全局最小割。那么就可以结束了（。但在算法运行过程中我们根本不知道它是不是。。）。。如果不是。。那么显然合并这两个点对解没有影响。。迭代 n-1 次。。

。。由于涉及 Union 操作、加之 prim 过程的 Extract 操作。。坊间 几乎都 开一个 vis 数组。。
。这里为了编码方便 + 常数优化。。。仍然手写邻接表。。

inline void del(int x){prd[suc[prd[x]] = suc[x]] = prd[x];}
inline void rsm(int x){prd[suc[suc[prd[x]] = x]] = x;}

#define hd suc[0]
void Extract(int &s){
s = hd; REP_L(i, suc[s], suc) if (cut[i] > cut[s]) s = i;
}

void Prim(){
REP_L(i, hd, suc) cut[i] = 0; tt = 0; DO(nn-1){
Extract(s); del(s); tmp[tt++] = s;
REP_L(i, hd, suc) cut[i] += C[s][i];
}
Extract(t); DWN(i, tt, 0) rsm(tmp[i]); del(t);
}

quosi-prime 过程。。前 n 次提取 s。。最后一次提取 t。。。注意在 Prim 结束。。删除 t 点之前。。要先 resume 回去。。。）
。。。

回到原问题。。 。并查集似乎不可避免。。

每次用 Stoer_Wagner() 求出全局最小割（实际上不一定需要是最小割。中间时刻的任意一个 < K 的割都行、。）。
。如果割 >= K。。那么当前图已经 k-edge connected。。。break 掉。
。否则。。 将图一分为二。。两边递归下去即可。。。。
