C:

注意到 k 的范围很小。。先不考虑 r 。。。
那么每个修改操作。。就是一个后缀增加一列组合数。。。。

示意图。。。

1
1 1
1 2 [1]
1 3 [3] 1
1 4 [6] 4 1
1 5 [10] 10 5 1

我们希望对 l 的影响可以延续到 l+1, l+2。。。
。。。这个影响可以二维 DP 维护。。。也就是和预处理组合数本身的过程相似。。。

设 A[lv][i] 为
。。。第 lv 层，i 的值。。
。。这里 lv 的含义是。向后产生作用的延迟标记。。。。
。。。0 表示立即产生作用。。1 表示会从下一个数开始产生作用。
。。。那么对于作用在第 l 个数的第 k 列修改操作。。我们把 lv = 0..k 的部分都打满标记即可。。。

最后再一次 DP 。。。

const int N = int(1e5)+1, M = 102;
Int Binom[N+M][M], A[M][N], A0[N];
int n, m;

int main(){

#endif

REP(i, N){Binom[i][0] = 1; REP_1(j, min(M-1, i)) Binom[i][j] = Binom[i-1][j-1] + Binom[i-1][j];}
RD(n, m); REP(i, n) RD(A0[i]); DO(m){
int l, r, k; --RD(l, r, k);
REP(lv, k+1) A[lv][l] += Binom[k][k-lv], A[lv][r] -= Binom[k+r-l][k-lv];
}

REP_2(i, lv, n, M-1) A[lv][i+1] += A[lv][i] + A[lv+1][i];
REP(i, n) OT(A[0][i]+A0[i]);
}









D:

枚举每一行 i。。
dp[l][r]：
表示当前行，[l, r] 这段区间最大向上可以延伸到的位置。。

显然可以区间 dp。。。
checkMax(dp[l][r], dp[l+1][r], dp[l][r-1]))。。

那么没有在这个 DP 方程中包含到的转移就是。。。
A[i][r] 和目前为止所有第 l 列（包含第 i 行）的元素，以及
A[i][l] 和目前为止所有第 r 列（包含第 i 行）之间的判重。。。

设 p[ a ][ j ] 表示第 j 列元素 a 最晚出现的行号。（这个空格和值域相关。。这题里刚好可以开下。。否则的话注意 hash。。）
。。。这样初始化和转移都是 O(n2) 的。。。
总的复杂度是 O(n3)。。。

const int N = 401;

int A[N][N], dp[N][N], pre[N*N][N];
int n, m;

int main(){

#ifndef ONLINE_JUDGE
freopen("in.txt", "r", stdin);
//freopen("out.txt", "w", stdout);
#endif

RD(n, m); REP_2(i, j, n, m) RD(A[i][j]);

#define p pre[A[i][j]][j]
int z = 0; REP(i, n){
REP(j, m) checkMax(dp[j][j], p), p = i+1, checkMax(z, i-dp[j][j]+1);
REP(l, m) FOR(r, l+1, m) checkMax(dp[l][r], max(pre[A[i][l]][r], pre[A[i][r]][l]));
REP(len, m) REP(l, m-len){
int r = l + len;
if (l != r) checkMax(dp[l][r], max(dp[l+1][r], dp[l][r-1]));
checkMax(z, (len+1)*(i-dp[l][r]+1));
}
}

OT(z);
}
